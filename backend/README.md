# HireMePlz Backend

Express + Prisma API for the HireMePlz MVP.

This service provides authentication, profile management, experiences, story library, application tracking, autofill suggestions, and job recommendation APIs.

## Tech Stack

- Node.js 22
- TypeScript
- Express
- Prisma
- PostgreSQL
- Zod
- JWT (`jsonwebtoken`)
- OpenAI SDK (optional, for richer autofill answers)

## What This Backend Does

- Auth: register, login, and current-user session validation
- Profile: manage personal info and education records
- Experience: CRUD for work/project experience
- Story Library: CRUD for STAR-style story items
- Applications: create and track job application status
- Autofill: generate field suggestions for extension/web forms
- Jobs: expose recommendation results generated by worker

## Prerequisites

- Node.js 22+
- npm 10+
- PostgreSQL running locally (or a reachable database URL)

## Environment Variables

Create a root-level `.env` file (recommended from project root):

```bash
cp .env.example .env
```

Required and commonly used values:

```env
DATABASE_URL="postgresql://postgres:postgres@localhost:5432/hiremeplz?schema=public"
JWT_SECRET="replace-with-a-long-secret"
OPENAI_API_KEY=""
OPENAI_MODEL="gpt-4o-mini"
PORT="4000"
```

Notes:

- `OPENAI_API_KEY` is optional. If empty, autofill open-ended answers fall back to template-based generation.
- `backend/src/config/env.ts` loads `../.env` first, then process env, so running from workspace root works out of the box.

## Local Development

From repository root:

```bash
npm install
npm run db:generate
npm run db:push
npm run dev:backend
```

Or from `backend` only:

```bash
npm install
npm run db:generate
npm run db:push
npm run dev
```

The server starts at:

- `http://localhost:4000`
- Health check: `GET /api/health`

## NPM Scripts (`backend/package.json`)

- `npm run dev` - Start dev server with `tsx watch`
- `npm run build` - Compile TypeScript to `dist/`
- `npm run start` - Run compiled server
- `npm run lint` - Type check (`tsc --noEmit`)
- `npm run db:generate` - Generate Prisma client
- `npm run db:push` - Push Prisma schema to database

## API Overview

Base prefix: `/api`

- `GET /health`
- `POST /auth/register`
- `POST /auth/login`
- `GET /auth/me`
- `GET|PUT /profile`
- `POST|PUT|DELETE /profile/educations/*`
- `GET|POST|PUT|DELETE /experiences/*`
- `GET|POST|PUT|DELETE /stories/*`
- `GET|POST /applications`
- `PATCH /applications/:applicationId/status`
- `POST /autofill/suggestions`
- `POST /autofill/record`
- `GET /jobs/recommendations`

Most routes require `Authorization: Bearer <token>` except health and auth register/login.

## Data Model Highlights

Defined in `prisma/schema.prisma`:

- `User` + `Profile`
- `Education`
- `Experience`
- `StoryItem`
- `Application`
- `Job`
- `JobMatch`

`Job` and `JobMatch` are mostly populated by the `worker` service, while this backend exposes recommendation queries.

## Development Notes

- Error handling is centralized in `src/middleware/error-handler.ts`.
- Request validation is done with Zod in each module router.
- Current architecture is route-centric (business logic inside route files), which is simple for MVP and easy to iterate on.
