CORE REQUIREMENTS

Docker, Multi-Container & Stateful Design

11 Containers Working Together

frontend backend worker worker-llm postgres redis prometheus grafana loki promtail db-backup
  • Each service has its own Dockerfile
  • Docker Compose for local one-command startup
  • Health checks enforce startup ordering

How State Survives Restarts

  • PostgreSQL stores all user data, jobs, applications
  • Runs as a StatefulSet with a 5 Gi PVC
  • Data persists across pod restarts and redeployments
  • Redis as BullMQ message broker for async tasks
  • Daily backup to DigitalOcean Spaces
kind: StatefulSet volumeClaimTemplates: - spec: accessModes: ["ReadWriteOnce"] storage: 5Gi
CORE REQUIREMENT

Kubernetes Orchestration

Deployed on DigitalOcean

  • Kustomize manages all K8s manifests
  • Backend: 2 replicas with RollingUpdate
  • InitContainer runs DB migration on deploy
  • Readiness & liveness probes on every service
  • Resource requests & limits defined

Ingress Routing (3 domains)

  • hiremeplz.info → Frontend
  • api.hiremeplz.info → Backend API
  • grafana.hiremeplz.info → Grafana

Monitoring & Observability

Custom Alert Rules

  • 5xx error rate > 5%
